Conditions

With conditions you can filter the users you want to target. The available fields include: user id, username, group id, number of posts, registration date, last active, ..., and even custom profile fields. You can add as many conditions as you wish by clicking the add button in the last row of the conditions table. You can also remove any unwanted condition by clicking the remove button on each condition row.

Each condition has three parts:

Field: the name of the field that holds the information you want to check. Every field has a specific type that can change the context of the test operator and the value.
  • User ID: number
  • Username: text
  • E-mail: text
  • Number of posts: number
  • Usergroup (ID): number
  • Registration date: date
  • Last active: date
  • Away status: boolean (true/false)
  • Return date: date
  • Reputation: number
  • Registraton IP: ip address
  • Last used IP: ip address
  • Time spend online: number (of seconds)
  • Language: text
  • Custom fields are all treated like text

Test: the operator you want to use to test a chosen field for the given value. Tests have a different purpose for the different field types.
  • number
    • equal / not equal: test if field has the same numeric value as the given value.
    • is null / is not null: test if the field has the value of NULL or not.
    • is empty / is not empty: test if the field is empty or not (= no value).
    • lower than / greater than: test if the value of the field is lower or greater than the given value.
    • in / not in: test if the value of the field is included or not included in the give list of values (comma separated).
    • like / not like: no significant purpose for a number.
  • text
    • equal / not equal: test if field has the same value as the given value.
    • is null / is not null: test if the field has the value of NULL or not.
    • is empty / is not empty: test if the field is empty or not (= no value).
    • lower than / greater than: test if the value of the field is lower or higher in the alphabet than the given value.
    • in / not in: test if the value of the field is included in the give list of values (comma separated). If you would like to include a comma in your value you need enclose the value in quotes.
    • like / not like: test if field value matches the given pattern. You can use the asterisk sign (*) for a wildcart. For example: abc*e
  • date
    • equal / not equal: not really helpfull for dates. Use lower than / greater than instead.
    • is null / is not null: test if the field has the value of NULL or not.
    • is empty / is not empty: test if the field is empty or not (= no value).
    • lower than / greater than: test if field is before or after the given date value.
    • in / not in: no significant purpose for a date.
    • like / not like: no significant purpose for a date.
  • boolean
    • equal / not equal: test if field has or hasn't the same value as the given value.
    • is null / is not null: test if the field has the value of NULL or not.
    • is empty / is not empty: test if the field is empty or not (= no value).
    • lower than / greater than: no significant purpose for a boolean.
    • in / not in: no significant purpose for a boolean.
    • like / not like: no significant purpose for a boolean.
  • ip address
    • equal / not equal: test if field has or hasn't got the same ip as the given ip.
    • is null / is not null: test if the field has the value of NULL or not.
    • is empty / is not empty: test if the field is empty or not (= no value).
    • lower than / greater than: test if the ip stored in the field is lower or greater than the given ip.
    • in / not in: test if the ip stored in the field is in the given list of ip addresses (comma separated).
    • like / not like: test if the ip stored in the field matches the given pattern. You can use the asterisk sign (*) for a wildcart.

Value: the value you want to test for. Some field types can have different types of values.
  • number:
    • equal / not equal: a numeric value (for example: 1 or 2 or 999)
    • is null / is not null: no value
    • is empty / is not empty: no value
    • lower than / greater than: a numeric value (for example: 1 or 2 or 999)
    • in / not in: a list of numeric values (for example: 1,2,3,999).
    • like / not like: not applicable
  • text:
    • equal / not equal: a string value (for example: abcd or efgh)
    • is null / is not null: no value
    • is empty / is not empty: no value
    • lower than / greater than: a string value (for example: abcd or efgh)
    • in / not in: a list of string values (for example: abcd,efgh,\"abcd,efgh\").
    • like / not like: a pattern (for example: abc*fgh)
  • date:
    • A date can have two types of format:
      • absolute date: a date formatted as dd-mm-YYYY or mm/dd/YYYY. Not really helpfull because the reminder will only work for one specific date.
      • relative date: a textual representation in English of a relative date. For example: -1 week (= one week ago), +2 week (= 2 weeks from now), today (= today). Please read the strtotime() documentation for more information about supported formats.
    • equal / not equal: a date value (for example: 02-12-2011)
    • is null / is not null: no value
    • is empty / is not empty: no value
    • lower than / greater than: a date value (for example: +1 month)
    • in / not in: a list of dates (for example: 02-12-2011,04-12-2011).
    • like / not like: not applicable
  • boolean:
    • equal / not equal: 0 or 1
    • lower than / greater than: not applicable.
    • is null / is not null: no value
    • is empty / is not empty: no value
    • in / not in: a list of dates not applicable.
    • like / not like: not applicable
  • ip address:
    • equal / not equal: an ip address (for example: 111.111.111.111)
    • is null / is not null: no value
    • is empty / is not empty: no value
    • lower than / greater than: an ip address (for example: 111.111.111.111)
    • in / not in: a list of ip addresses (for example: 111.111.111,222.222.222.222).
    • like / not like: a pattern (for example: 111.111.*)
